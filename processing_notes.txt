                            # Hourly weather
########################################################################

\\ARS-DATA\Archive\HarmelExit\riesel\TxBLPhydro\I. historical data\weather\hourlyweather2009.zip

The .txt files in these two folders are from 2001 - 2009.
From 2001 - 2003 There is no vapor pressure data 
From 2004 - 2009 Vapor pressure is added and headers differ from 01-03.

Attempts were made to group the first 3 rows of the files to obtain
the same headers.
After discussions with supervisor it was determined that the ICASA 
Data Standards would be used.
The dataframe structure was coded accordingly.

Upon further inspection of the data it was noticed that there are
what appears to be extra values in the same columns which has thrown
the next values in sequence off. 

Code was developed to identify these errors and return the row # 
corresponding to the row while viewed in VS Code.
Further development was made to print the labels and row values
in a orgainised fashion.

Code was developed to isolate the rows prior and after the occurance
of the extra values for the 2001 - 2003 files.

Code was developed to read the 2004 - 2009 files and identify errors
in the values.

By utilizing the length of a row as well as reading the values as 
floats:
for the 2004 - 2009 files -if there are extra values the row will be 
appended to a list of check_values. If the rows are fine they will be 
read into a pandas DataFrame.

For the dataframes created from both files the index corresponds to 
the index of the file when viewed with VS code.

Navigating the .txt files has shown that there are times that sensors
fail, mostly, the values will be -9999.00. Additional code will be
developed to identify these occurances.

                     Analysis of 2003
Reviewing each row of data and comparing the previous and next row led
to a trend appearing where RAIN contained a value of 1.00 and STMAX
would have a double value. This trend does not apply to every entry
a more in depth analysis will be provided.

THe rows of data were modified by removing the 1.00 and moving the 
values 1 cell left.While the modified RAIN value is still has an 
inconsistent value while comparing to the previous and next rows.
The STAVG | STMAX | STMIN values show more consistency. 
A STAVG2 was calculated from the modified values and was compared to
the STAVG if the data was modified.
The STAVG2 was more inconsistent when the STMIN values were negative.
It should be noted that the STAVG2 is calculated from the 
STMAX and STMIN, whereas reported STAVG values were calculated from
all the values within a day.

                    Analysis of 2004-2009
Reviewing the data in these files, it was noticed that several values
in multiple columns were -99999.00 or 99999.00 These are obvious
fails in the sensor. 
To handle these failures the rows containing either of these two
values were added to a new dataframe. For the readable dataframe (df)
these values were converted to "nan" float values.
There are only a few instances where the rows contained extra values
similiar to the extras found in the 2003 file. Apart from 2004 where
rows 4 - 35 measured data are all "na", except for max and min soil Temp
values. The "na" values will be converted to "nan" and "nan" will be
appended to rows 4 - 35 at the max and min soil temps.
